## Context

当前占位符语法 `{{group_name}}` 只能从预设 group 中选择，缺乏灵活性。用户希望能直接输入自定义内容（如端口号、文件名等）。

新的占位符语法：`{{意图[:提示语][|默认值]}}`

- `g` 意图：从指定 group 选择
- `i` 意图：弹出输入框手动输入

## Goals / Non-Goals

**Goals:**

- 支持 `g` 和 `i` 两种意图模式
- 统一的占位符解析流程
- 清晰的 UI 提示（提示语 + 默认值）
- 与现有 TUI 框架（Textual）集成

**Non-Goals:**

- 向后兼容旧语法（breaking change）
- 保存用户输入到配置文件
- i 模式关联 group 候选列表

## Decisions

### 1. 正则表达式设计

```python
# 新语法: {{意图[:提示语][|默认值]}}
# 示例: {{g|zellij}}, {{i:请输入端口|8080}}, {{g:选择容器|zellij}}

PLACEHOLDER_PATTERN = re.compile(
    r"\{\{(?P<intent>g|i)"
    r"(?::(?P<prompt>[^|]+))?"
    r"(?:\|(?P<default>[^}]+))?"
    r"\}\}"
)
```

**Rationale**: 使用命名捕获组提高可读性，`[^|]+` 和 `[^}]+` 确保正确处理包含 `|` 的内容。

### 2. 解析器架构

```
┌─────────────────┐
│  extract_placeholders()  │
│  提取所有占位符           │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  parse_placeholder()     │
│  解析意图/提示语/默认值    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  resolve_placeholder()    │
│  g: 显示列表 / i: 显示输入框 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  replace_placeholder()    │
│  替换内容并继续            │
└─────────────────┘
```

**Rationale**: 分层设计，每个函数职责单一，便于测试和扩展。

### 3. 状态机设计

```python
class USelApp:
    # 模式状态
    _mode: Literal["searching", "resolving_g", "resolving_i"]
    _current_intent: str | None
    _current_prompt: str | None
    _current_default: str | None
```

| 模式          | 触发条件    | UI 显示             |
| ------------- | ----------- | ------------------- | ----------------------- |
| `searching`   | 默认状态    | 输入框 + 选项列表   |
| `resolving_g` | `{{g        | ...}}`              | 输入框提示 + group 列表 |
| `resolving_i` | `{{i:...}}` | 输入框提示 + 输入框 |

**Rationale**: 用状态机区分不同阶段的 UI 行为，避免 if-else 嵌套。

### 4. UI 布局

两种模式复用同一个容器，切换时更新内容：

```
┌─────────────────────────┐
│ [提示语]                 │  ← 输入框区域（动态）
├─────────────────────────┤
│  1. option 1            │
│  2. option 2            │  ← g 模式显示选项
│  3. option 3            │
└─────────────────────────┘

┌─────────────────────────┐
│ [请输入端口]             │  ← i 模式提示语
├─────────────────────────┤
│ [8080____________]       │  ← 输入框（带默认值）
└─────────────────────────┘
```

## Risks / Trade-offs

| Risk                           | Mitigation                     |
| ------------------------------ | ------------------------------ |
| 用户输入包含 `}}` 导致解析错误 | 正则使用 `[^}]+` 匹配非贪婪    |
| 嵌套深度过深导致栈溢出         | MAX_DEPTH=3 检查，提前退出     |
| i 模式回车空输入               | 强制检查，输入为空则报错退出   |
| group 不存在导致无选项显示     | 解析时检查，group 不存在则报错 |

## Open Questions

1. **i 模式是否需要 trim 用户输入？** 决定是否保留前后空格
2. **是否支持快捷键切换模式？** 如 `Ctrl+G` 切换到 g 模式，`Ctrl+I` 切换到 i 模式
